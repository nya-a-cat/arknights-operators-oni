# 代码审查与路线图（2026-07-15）

## 结论

`0.3.2-alpha.1` 已达到 GitHub prerelease 与 Steam Workshop Alpha 的技术发布条件。当前版本完成真实游戏加载、中英日干员名与重定向别名搜索、中英文设置界面、皮肤/模型联动、热切换、地面对齐、移动动画和缓存完整性验证。公开仓库排除约 170 MiB 的开发帧、游戏美术、Spine 资源、PRTS 网页构建产物和本地缓存。

本轮并行审查按渲染、动画、资源/文档三个边界进行。高优先级正确性问题已经修复；性能、国际化、干员语音、细粒度选择和复杂视觉增强保留为后续独立任务。

## 本轮已完成

## 后续施工：全干员备用资源中转

已选择 GitHub Release 作为 PRTS 故障备用源，并完成运行时代码、内存包回退测试和 GitHub Actions 分片生成器。稳定清单覆盖目录内全部 449 个干员、皮肤和模型；每个干员对应一个独立包，PRTS 主路径失败后按当前选择下载。全量资源由 GitHub runner 生成，本机只运行小 fixture。首个 `assets-v1.0.0` 快照仍处于“代码准备完成、等待确认上传”的状态，未把待生成资源描述成已发布能力。完整合同和发布验收见 `docs/github_release_asset_fallback.md`。

## 下一阶段优先级

以下条目均为后续候选功能，`0.3.2-alpha.1` 尚未实现。施工时每次只拆一个可验证功能，先确定资源与交互合同，再修改运行逻辑。

### P1：用户可见质量

1. **完整 i18n 资源化**：当前 Options 与 `Ctrl+F8` 界面已自动选择中文或英文，干员名已支持中英日搜索和语言优先显示。后续把运行时错误、下载状态与诊断迁移到 ONI `Localization`/`STRINGS` 资源，并增加更多界面语种。完成标准是各支持语种下的设置、错误提示和诊断均可实机验证。
2. **干员语音**：为当前干员提供可选语音语言、总开关、独立音量、试听按钮和事件触发。第一版只覆盖选中复制人、受伤/高压、死亡和低频工作反馈；移动、待机等高频事件默认不触发，使用全局并发上限、同复制人冷却和事件优先级避免多人同时说话。切换干员时停止旧语音，缺失语种或音频时静默降级。语音文件复用现有按需/永久缓存、临时文件、hash 和单文件体积限制。完成标准是至少两个干员、两种语言在试听、事件触发、热切换、重启离线命中和多人抢占下通过实机测试。
3. **每个复制人独立外观与语音**：全局默认继续存在，在复制人详情面板增加覆盖值。需要明确存档序列化、复制人删除后的清理、全局设置变更和不同干员声音并发语义。
4. **外观预览与语音试听**：设置窗口保存前显示当前皮肤/模型的静态或低成本动画预览，并提供单句语音试听；预览资源与游戏内资源共享缓存，关闭窗口后释放临时实例。
5. **自动模型选择**：基建模型用于日常移动/工作，正面或背面模型用于战斗/特定状态。先建立可复现的 ONI 状态到模型合同，再实现跨模型相位切换。
6. **下载/错误状态 UI**：在设置或复制人详情中显示下载进度、缓存命中、失败原因与重试入口，减少仅依赖 `Player.log` 的诊断成本。

### P2：个性化与管理

1. **收藏、最近使用与预设**：收藏常用干员/皮肤，显示最近使用项，并允许保存“干员 + 皮肤 + 模型 + 语音语言”预设；所有入口复用同一规范化选择逻辑。
2. **打印舱分配池**：允许为新复制人从收藏池顺序或随机分配干员，保留“沿用全局默认”选项；随机结果写入存档，重载后保持一致。
3. **缓存管理器**：按干员展示外观/语音缓存占用、最后使用时间和完整性状态，支持清理单个干员、清理按需缓存和重新校验；删除正在引用的资源必须被拒绝。
4. **诊断与一键自检**：导出 Mod 版本、ONI build、当前选择、资源版本、缓存索引摘要、最近一次错误和动画列表；默认排除存档内容、用户名和完整本机路径。
5. **目录更新与兼容提示**：新目录先生成 diff，显示新增/移除的干员、皮肤和模型；通过 schema/version 检查后原子替换，失败时继续使用上一份目录。
6. **Mod 冲突策略**：检测其他复制人外观覆盖 Mod，提供“本 Mod 优先、对方优先、禁用当前复制人覆盖”选择，并记录最终生效来源。
7. **版本化配置迁移**：为全局设置和未来的逐复制人数据建立显式 schema 迁移及回滚测试，防止改名或目录更新让存档中的选择丢失。

### P2：内容扩展（`arknights-oni` 仓库范围）

1. **类型化内容包框架**：把资源身份从当前干员外观扩展为 `operator`、`voice`、`furniture`、`enemy`、`effect` 五类；每类拥有独立 manifest、版本、体积统计和启用开关，底层继续复用下载、hash、租约和两种缓存策略。
2. **明日方舟基建与家具**：按家具套装或主题打包，把家具、墙面、地面、背景和可动装饰映射到 ONI 装饰建筑。第一阶段只替换视觉和装饰表现，保留 ONI 原建筑的尺寸、数值、占格、交互和自动化端口；验收重点是锚点、遮挡层、旋转/翻转、建造预览与拆除后的资源释放。
3. **敌人与生物外观**：先给 ONI 现有小动物或敌对实体提供可关闭的外观替换，映射待机、移动、攻击、受伤和死亡动画，同时保持碰撞体、寻路、AI、掉落和战斗数值。新增独立实体需要另开设计任务并先确定存档兼容和模拟负担。
4. **技能与环境特效**：把明日方舟特效绑定到明确的动画/游戏事件，覆盖攻击命中、工作完成、受伤、死亡和环境装饰。运行时使用对象池、并发上限、可视区域裁剪和质量档位；验证透明混合、双色、粒子/序列帧、暂停/倍速以及 20 个实体同时触发时的帧时间。
5. **内容包选择与依赖**：设置中按类别启用内容包，显示大小、来源、版本和依赖；禁用某包后停止新加载，当前引用释放后再清理，避免中途销毁正在显示的家具、敌人或特效。
6. **后续候选资产**：头像/复制人详情 UI、房间主题、加载画面和环境音可在上述四类稳定后评估。每类都先做小样本来源、格式、体积和运行时兼容检查。

### P1：资源一致性

1. **发布首个全干员备用快照**：运行手动 GitHub Actions 工作流，核对 449 个干员包、合并清单、体积报告和抽样 hash，发布前启用 immutable releases。代码与本地测试已完成，Release 尚未上传。
2. **bundle 原子提交**：atlas、skel 和全部纹理先进入 bundle 暂存区，全部校验后统一登记 Ready，避免 bundle 中途失败留下可见的半套资源。
3. **更早获取租约**：解析 bundle 时先保护已确认资源，降低容量维护与热切换并发时的删除窗口。
4. **永久策略专项验收**：下载后重启、断网命中、策略往返和手动清理都需要保存索引/磁盘证据。
5. **备用快照更新合同**：每次更新使用新 tag，先生成目录 diff 和资源 hash，再更新 Mod 固定引用；不覆盖已发布快照。

### P2：渲染与性能

1. 共享解码后的 atlas、纹理、骨骼数据；每个复制人只保留独立 Skeleton/AnimationState。
2. 为 20 个复制人记录托管内存、纹理内存和帧时间，按数据决定 mesh buffer 与材质缓存优化。
3. 验证 clipping、path constraint、多页、Additive/Multiply/Screen 与非黑双色模型的游戏内视觉一致性。
4. 给帧降级路径增加完整 Begin/Main/End 队列；当前实时 Spine 路径优先。

### P3：框架重估与多游戏内容方向

该里程碑是项目的确定性长期方向。当前阶段继续以明日方舟为第一套参考实现，通过真实干员、语音、家具、敌人和特效积累可验证的内容合同；框架边界以这些实现证据为依据。

1. **启动条件**：至少完成两类非同构内容包并通过实机验收，同时具备稳定的 manifest、缓存生命周期、选择 UI、事件映射、错误降级和配置迁移。达到条件后启动一次正式架构重估。
2. **候选分层**：将 ONI 状态/事件桥接、下载缓存、内容选择、包依赖和诊断能力整理为通用运行时；把 Spine、音频、序列帧或未来格式处理为格式适配层；把具体游戏资产映射保留在独立内容包中。
3. **第一参考实现**：明日方舟内容包继续承担默认示例、兼容性测试集和迁移基线，框架抽取后仍保持现有选择、缓存和存档数据可升级。
4. **其他游戏候选**：框架重估完成后再评估其他游戏内容包，`BanG Dream!` 作为示例候选之一。评估内容包括可用资产格式、角色与动画状态映射、语音组织、UI/场景资源、单包体积和 ONI 运行时成本。
5. **仓库边界重估**：根据通用核心与内容包的独立发布需求，决定继续单仓库、拆分核心仓库与内容包仓库，或建立共享 SDK；结论必须带迁移方案、版本策略和现有 `arknights-oni` 用户影响分析。

## 干员语音资源技术门槛

- PRTS 在审计时提供独立的[干员语音分类](https://prts.wiki/w/分类:干员语音)，语音记录页包含中文、日文、英文和韩文文本，并提供播放/下载入口。这证明功能具备资源发现入口，具体音频 URL、格式、大小、版本字段和跨域行为仍需单独探针确认。
- 语音功能进入施工前先确认资源 URL、格式、大小、版本字段、跨域行为和 ONI 运行时解码能力，并继续采用用户主动启用的逐文件获取方式。
- GitHub 仓库继续排除语音文件。第一版采用用户主动启用、运行时按当前干员和语种获取、逐文件缓存的路线，不生成全量语音包。
- 音频下载不得引入超过 100 MB 的单项依赖；实际实现前先统计两个代表性干员的格式、单句/整套体积和 ONI 可直接解码能力。

## 发布边界

- GitHub 仓库名称使用 `arknights-oni`，为干员、语音、基建家具、敌人和特效保留扩展空间。
- 当前显示名称为 `Arknights Operators（明日方舟干员）`。
- `AmiyaDuplicantMod` DLL、命名空间、目录和 `staticID` 暂时保留，避免破坏已有 Options 与 ONI 模组记录。
- 当前版本按全局选择应用到所有复制人。
- Alpha Mod 发布包只包含 DLL、PLib、配置清单、轻量目录和来源/许可说明；独立的资源 Release 按干员提供故障备用包，运行时只在主来源失败后获取当前干员。
- GitHub Release 与 Steam Workshop 使用同一版本号和同一内容包，便于核对 SHA-256 与回滚。
- 原创代码当前未授予额外开源许可证；第三方组件分别遵守仓库内许可文本。
- GitHub 源码仓库不包含游戏动画或图片。运行时优先使用 PRTS，失败时使用固定版本的项目 Release；两个冷来源同时不可用时仍会加载失败。
